SFExplorer v0.6 (beta) 1999.04.21

SFExplorer is a windows application for users of SoundFont compatible devices (such as SBLive, AWE32, etc.).

SFExplorer lets the user browse and preview (using MIDI) the SoundFonts currently loaded by the SoundFont compatible device, as well as load new SoundFonts and delete SoundFonts from memory. SFExplorer can also be set up as the default viewer for SoundFont files (*.sf2) for one-click preview.

Version 0.6 (beta) adds the possibility to browse directories on disk or CDs. This feature is great for quick access to all your SoundFonts, and for browsing SoundFont CDs

Version 0.6 (beta) is fully functionable, but lacks a lot of features and may have some errors. The main purpose of releasing it before it is finished is to get feedback from users. So please mail me and tell me about your experiences (good or bad) with this program.

Installation

Download and install Creative Labs' SoundFont Management System if you don't have it allready.

  1. Download and unzipl SFExplorer v0.5 (beta).
  2. Run Setup.exe.
  3. Download SFExplorer v0.6 (beta) and copy the executable (SFExplorer.exe) to the directory where you installed SFExplorer v0.5 beta (replacing the older version).

Alternatively (if you have problems getting the installer to work), you can download just SFExplorer v0.6 beta and SFExplorer Extras. Copy the executable to a directory of your choice and try to run it. If it complains about missing or too old files, copy files from SFExplorer Extras to your \Windows\System directory.

Documentation

Sorry, but you're pretty much on your own. The up-side is that the program is really not very complicated. Just remember to try the right mouse button.

If you should suddenly get problems starting SFExplorer (for instance after installing new soundcards, MIDI devices, etc), try deleting the file sfx.ini.

Version History

1999.04.21 Version 0.6 beta

  • Now features browsing of soundfonts stored on disk or CD as well.
  • 'Fixed' MIDI controller bug (program would crash if a MIDI CC message was sent to it) by skipping all MIDI messages besides note-on and note-off. I'll have to do some more debugging to find out why SoundFont devices doesn't like my CC messages.
  • Error reporting is better.
  • Fixed several minor bugs.

1999.03.07 Version 0.5 beta

  • The first public release. Allows browsing of loaded soundfonts and preview using MIDI. Several severe bugs were discovered after this release. User feedback is great!
